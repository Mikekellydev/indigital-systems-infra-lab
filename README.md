# INDIGITAL Systems Infrastructure Engineer Lab

This repository demonstrates the daily responsibilities of a Systems Infrastructure Engineer, including:

- Linux administration  
- Automation with Ansible  
- Virtualization with Proxmox + Terraform  
- Monitoring with Prometheus + Grafana  
- Security hardening and vulnerability management  
- Documentation, runbooks, and incident response  

The project is structured to reflect real-world engineering workflows and operational practices.

---

## Sections

### 1. Ansible
Automation for:
- Linux baseline configuration  
- NGINX deployment  
- MySQL provisioning  

### 2. Proxmox IaC (Terraform)
Infrastructure-as-code for VM provisioning using cloud-init templates.

### 3. Monitoring
Prometheus configuration, alert rules, and Grafana dashboards.

### 4. Security
CIS hardening, vulnerability scan summaries, and certificate automation.

### 5. Documentation
Runbooks, incident response examples, change management, and architecture diagrams.

---

## Purpose

This lab environment demonstrates readiness to step into a Systems Infrastructure Engineer role by showing:

- Technical depth  
- Operational maturity  
- Automation mindset  
- Security awareness  
- Clear communication
